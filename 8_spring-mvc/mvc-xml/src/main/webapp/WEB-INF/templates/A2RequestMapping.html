测试RequestMapping的用法

测试@value属性，可以进行多种访问路径的访问

  1. 通过"/a?a"访问(?代表任意单个字符)
  2. 通过"/ab"访问"/a*b"(*代表0个或多个任意字符)
  3. 通过"/a/b"访问"/a/**"，匹配任意路径

测试@method属性，符合访问类型才能访问，浏览器会将不属于post的请求类型，视为get请求

get访问

测试@param属性，必须所有的参数符合，才能匹配

params = {"param1", "!param2", "param3=c", "param4!=d"}，必须有param1、必须有param3 = c、必须没有param2、必须有param4且 != d

param1： a b
param2： a b
param3： c d
param4： c d
